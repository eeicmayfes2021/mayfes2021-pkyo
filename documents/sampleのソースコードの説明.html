sampleのソースコードの説明

はじめに

ここで、手を加えるべきファイルはindex.html、style.css、index.jsなのでこの3つについて説明します。

index.htmlとstyle.css

index.htmlは、
<div id="phaserDiv"></div>
と
<div id="blocklyDiv"></div>
だけ知っていれば良さそうです。
style.css内で
phaserDivとblocklyDivのwebブラウザ上での座標と大きさを設定している、という程度の認識で大丈夫です。

sampleでは
<xml id="toolbox" style="display: none"> <block type="while"></block> </xml>
ていうのがあって、ここで後述するweb上に表示するブロックを指定しています。
通常ならブロックは定義したものを指定する必要があるんですが、例外もあります。
control_ifとかcontrol_whileなどのブロックはBlockly Developer Tools上ですでに用意されているらしく、それらを指定して定義なしでつかうこともできます。

index.js

import hoge

これは深く考えなくていいです。ライブラリだけでなく、pngファイルとかもimportできるんだ〜程度でOKです。
exportとかいう機能を使えば他のJSファイル内で定義したクラスとかもimportできるらしい。

var config …

ゲーム表示の設定部分です。
これをPhaser.Game関数に渡すことによってゲームの骨格ができると考えて良さそうです。

configの中身

type: よくわかりません。リファレンスを読むとPhaser.AUTOでOKと書いてあったのでこうなってます。

widthとheight: ゲームの画面の大きさを定義しています(画面は長方形です)。このパラメータをいじると、実際に画面の大きさが変化するのがわかると思います。

parent: これはおそらくゲーム画面をおく場所を指定しています。htmlファイル上で定義したphaserDivという場所に左上の頂点が来るようにしています。

physics: 物理演算部分だと思います。arcadeという設定に従うということにして、arcade内を見ればわかりますがx軸、y軸方向の重力を0にしています(今回製作するゲームに重力は不要なので)。

scene: ゲームのシーンを定義します。リファレンスを読みきってないのでまだわかりませんが、おそらく上から順に実行されます。
それぞれのsceneで行う関数を以降で実装する形になります。ここではsceneのpreload, create, updateにそれぞれ同じ名前の関数を割り当てています。

render: よくわからないけど、transparentとすると嬉しいらしい。

const blocklyDiv = …

html上で定義しているブロックを組み立てる領域を変数に格納しています。
一行下のblocklyDiv.style.left = config.width;
でその領域が、ゲーム画面の右端と繋がるように座標を調整しています。(ここ怪しい、調整してくれなさそうなので、CSSを直接いじりましょう)

var game = …

ゲームの挙動を担当するPhase.Gameのインスタンスを作成しています。

function preload()…

configで定義したsceneの一つpreloadに対応した関数です。
ここではステージやキャラクターの画像を取得しています。

function create()…

取得した画像の情報を実際にオブジェクトに貼り付け？て表示しています。
他にも、キャラクターの初期位置を設定しています。

preloadとcreateでやってることの詳しい内容は、「マップ作成やプレイヤー配置の色々.html」を読んでください

function update()…

まぁupdate部分です。
ブロックを組み立てて実行したときに関数を呼び出したり、キャラの座標を更新したりしています。

Blockly.Blocks[hoge] = …

ブロックの定義をしています。
ここでブロックを定義すると、ゲーム上に表示できるようになります(xmlファイル上に対応する名前のblockを書く必要はありますが)。

Blockly.Javascript[hoge] = …

これは上で定義したブロックの機能を書く部分です。
対応したhogeのブロックの機能なので、ブロックを定義した分だけここも実装する必要があります。

ブロックの作成方法の詳しい内容は「Blocklyのすべて.html」を読んでください。

tryMove()…

「move」ブロックが実行された場合に使われる関数を定義しています。
第二引数に渡されたindexに対応する方向へ一マス移動します。

var options = …

これは、ブロックをブラウザに表示する際の様々なオプションです。
以下に、重要でわかりやすいものだけ抜粋します。
toolbox: 表示するブロックたちを渡します(昨年度はxmlファイルを渡していたが…)
sampleでは、document.getElementById('toolbox')によって、html上のtoolboxというidを割り当てた部分を渡しています。

trashcan: ゴミ箱を表示するかどうかを決めます。

var workspace = Blockly.inject(…)

injectの第一引数にブロックを表示する場所(ここではhtml上で定義したblocklyDiv)、第二引数にその場所に付け加える色々なオプションを定義したものを渡します(本来はここにさっき定義したoptionsを渡します)。

これをすることで、画面にブロックが現れます。

runcode(…)

生成された命令を実行します。
やっていることは各ブロックが生成して溜まった命令を順次実行している感じです。命令がなくなるまで実行されます。

LoadBlocksandGenerateCommand()…

画面上の緑のボタンをおすと実行されます。
この関数が実行されるとspace上のブロックがすべてコード化され(上から順番にコード化されます)、codeという変数に文字列として渡されます。この文字列をeval関数によって実行している感じです。(eval関数はよくないらしいです)
