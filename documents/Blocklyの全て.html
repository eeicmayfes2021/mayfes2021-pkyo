Blocklyの全て

はじめに

Blocklyのリファレンスを読んで得られた知見(というよりもプログラミング教室の開発に必要そうなこと)をまとめています。

これよりもっと詳しく知りたいって人は、リファレンスを読んでね(そこに全てが載ってます)。

なお、JavaScript上で作成することを想定しています。

Blocklyブロックの作り方

↓こういう感じのブロックが作れます。

JavaScriptでは







Blockly.Blocks['ブロックの名前'] = {
    init: function(){
        hogehoge
    }
    hogehoge
}

っていう感じに定義します。

一から自分で作るのは結構厳しいので、Blockly Developer Toolsを頼りましょう。

↑こんなサイトで、左側の部分でブロックを組み立てると、Previewにそのブロックの外見、Block Definitionにさっき紹介した定義のコードが自動生成されます。

どうブロックを組み立てたらどんなブロックができるのかは習うより慣れろって感じなのでここには書きませんが、一つ例を挙げるとこんなんができます。

こうして自分の望み通りの機能を持ったブロックができたら、Block Definition部分のコードをコピペしてJSファイルに貼りましょう。
あとはxmlファイル(多分まだhtml上に書いてます)にこんな感じ↓に追加すればweb上に表示されるようになります。

    <block type="hoge"></block>

上の画像のブロックならhoge = whileです。

一応この画像のブロックの定義文もここに貼っときます。














Blockly.Blocks['while'] = {
  init: function() {
    this.appendValueInput("key")
        .setCheck(null)
        .appendField(new Blockly.FieldLabelSerializable("この中をくりかえします"), "string")
        .appendField(new Blockly.FieldDropdown([["a","0"], ["b","1"], ["c","2"]]), "number");
    this.appendStatementInput("state")
        .setCheck(null);
    this.setColour(230);
 this.setTooltip("");
 this.setHelpUrl("");
  }
};

blocklyブロックの機能の実装

ここまでで、ブロックの定義の作り方を書いたんですが、今のところ中身がないのでただのブロックです。
ここでは中身の書き方を例にあげた"while"ブロックをもとに説明します。

Blockly Developer toolsのBlock Definition欄の下にGenerator stubという欄があります。
実はブロックの機能の雛形を自動生成してくれていて、ここを埋めるだけで中身ができます。

雛形↓









Blockly.JavaScript['while'] = function(block) {
  var dropdown_number = block.getFieldValue('number');
  var value_key = Blockly.JavaScript.valueToCode(block, 'key', Blockly.JavaScript.ORDER_ATOMIC);
  var statements_state = Blockly.JavaScript.statementToCode(block, 'state');
  // TODO: Assemble JavaScript into code variable.
  var code = '...;\n';
  return code;
};

ブロックのやってること

普通のプログラミングの関数みたいに、引数を受け取って何かを出力しています。
引数は何個やねんってなるかもしれませんがそれはブロックの種類によります。

ただ引数自体は3種類だけです。例の"while"ブロックがちょうどその3種類の引数を受け取るタイプなので、それで説明します。

3種類の引数を受け取る時にはそれぞれ別の関数を用いる必要があって、以下の3つの関数を用います。

Blockly.JavaScript.getFieldValue
Blockly.JavaScript.valueToCode
Blockly.JavaScript.statementToCode

それぞれ以下の画像のように対応しています。

getFieldValue

"while"ブロックを例に説明しますが、定義のコードを見ると

.appendField(new Blockly.FieldDropdown([["a","0"], ["b","1"], ["c","2"]]), "number");

となっています。ここでの"number"によってここの引数の値を取得します。
具体的には

Blockly.JavaScript.getFieldValue("number");

とすると、値が取得されます。
ただし、得られる値はリストの右側、つまり今回は"a"なら"0"、“b"なら"1”、"c"なら"2"です。
おそらく文字列しか取得できないので、注意してください(要出典)。

valueToCode

画像のValue部分に差し込まれた他のブロックの出力を入力として受け取ります。

ただし、後述しますが各ブロックの出力はコードを文字列化したものなので、文字列が得られます。

取得例

Blockly.JavaScript.valueToCode(block, 'hoge', Blockly.JavaScript.ORDER_ATOMIC);

アタッチされたブロックの名前"hoge"に該当するJSのコードの文字列が得られます。
何もアタッチされていなかったら"0"が得られるらしいです。

なお、コード内の四則演算の順番などを考慮する場合は、それに対応する順番を指定しなければいけないみたいですが、今回の開発では使わないと思うのでORDER_ATOMICとしてOKだと思います。

statementToCode

基本的にvalueToCodeと同じです。
上の画像の赤文字でstatementToCodeと書かれた場所に入れられたブロックをコード化したものの文字列を取得します。

取得例

Blockly.JavaScript.statementToCode(block, 'hoge')

ここのhogeには挿入されているブロックの名前が入ります。

各引数をまとめて出力する

雛形のコードを見てみると、codeという変数を定義してそれを返り値にしています。



  var code = '...;\n';
  return code;

このcodeがこのブロックの挙動を表すJSのコードの文字列になっています。

上の"while"ブロックはややこしいので簡単のためにこのブロックで説明します。

「1マス進む」のブロックのstatementToCode関数による返り値がmoveという変数に格納されているとすると、このブロックの挙動はJSで書くと




    while(true){
        move
    }

なので、

    code = 'while(true){' + move + '}\n';

または

    code = `while(true){${move}}\n`;

となります。
あとはこれをreturnすればブロックの中身は完成です。
文字列自体がJSの文法を満たす必要があることに注意してください

ブロックを組み立てたあと

実際のゲームではこの画像の右側でブロックを組み立てます。

ブロックを組み立てる→実行ボタンを押す→キャラが動くという感じなので、組み上がったブロックをJSのコードの文字列に直す必要があります。

これは

    var code = Blockly.JavaScript.workspaceToCode(workspace);

でできます。

Phaser.Game内の関数(おそらくupdate関数)でこれを行って、文字列化したコードをeval関数(これはセキュリティ(グローバル変数を書き換えてしまうらしい)とか実行速度に問題があるようで、Function関数が良いらしいです)でコード化して云々って感じです(説明放棄)。
